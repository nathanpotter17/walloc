LAYER_W: SUBMODULE_001: WALLOC

WebAssembly Memory Allocator

This page loads the WALLOC library from LAYER W for memory management in WebAssembly.
Memory chunks are 64KiB, the max size of a WASM Page, however, the allocator can accept an arbitrary number of bytes above the minimum of 1 page. Total maximum allocation size is 65,536 pages.
(2^16 pages is 2^16 * 64KiB = 4GiB)

When do you need memory?

You must instantiate WebAssembly.Memory only if:

  • The module imports memory from the environment.
  • The module defines memory internally (and uses it).
  • Your runtime or compiled code (e.g., from C/C++ via Emscripten) expects it for heap/stack.
  • If you're manually writing WAT (WebAssembly Text) or compiling from minimal Rust/AssemblyScript, you can skip memory declaration entirely.
